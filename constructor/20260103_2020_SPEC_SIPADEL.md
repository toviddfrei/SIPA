📊 ESQUEMA DE DATOS

1. Situación Actual (El "Hoy")

* Actualmente, el arranque es rígido:
  * El EnvironmentManager tiene una lista fija de carpetas en el código.
  * Si falta una, el sistema se detiene (Fail-Safe).

Problema: Cada vez que añadimos un módulo (como SIPAdoc), tenemos que modificar el código del guardián manualmente.

1. Propuesta: Auditoría Progresiva (El "Mañana")

   * El objetivo es que el sistema sea autónomo y dinámico:
   * El Manifiesto: Crearemos un archivo JSON/Dict que define cómo debe ser SIPA (Carpetas, archivos críticos, permisos).
   * El Escaneo: El guardián lee ese manifiesto y verifica la salud de todo el árbol.
   * Evolución: Si mañana creas FHS-CyberAudit, solo añades una línea al manifiesto y SIPA ya sabe que debe protegerlo.
