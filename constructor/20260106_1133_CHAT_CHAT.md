## 🎯 OBJETIVO PEDAGÓGICO
Este documento define la frontera entre el **Código Fuente** (Lógica reproducible) y el **Conocimiento del Sistema** (Datos privados, evidencias y aprendizaje). La pedagogía aquí reside en entender que en un sistema con IA, el "Ruido" (cachés, bases de datos temporales) debe ser filtrado para que el aprendizaje sea limpio y seguro.

## 🧠 LÓGICA DE DISEÑO
Para la versión 0.2.5.0, se ha decidido que el archivo `.gitignore` no sea una simple lista, sino un **mecanismo de defensa**. Se basa en tres pilares:

1.  **Protección de la Propiedad Intelectual:** Los archivos en `data/knowledge/*.md` son el "cerebro" local del sistema. No deben salir del entorno local de Daniel Miñana.
2.  **Integridad de la Caja Negra:** Los archivos de la base de datos SQLite (`.db`, `.db-journal`) contienen la telemetría de uso real. Su exclusión evita conflictos de sincronización y filtraciones de actividad.
3.  **Seguridad CyberAudit:** Bloqueo estricto de cualquier archivo en `data/evidences/`. Este módulo maneja JPGs y PDFs sensibles que forman la trayectoria profesional.

## 📊 ESQUEMA DE DATOS / ESTRUCTURA DE EXCLUSIÓN
El fichero oficial aplicado en esta versión sigue el siguiente esquema jerárquico:

| Nivel | Categoría | Objetivo de Exclusión |
| :--- | :--- | :--- |
| **L0** | Python Runtime | Evitar basura técnica (`__pycache__`, `venv`). |
| **L1** | Persistencia | Blindar el archivo `sipa_kernel.db` y temporales. |
| **L2** | Conocimiento | Proteger los artefactos `.md` de entrenamiento IA. |
| **L3** | Evidencias | Aislamiento de documentación profesional (JPG, PDF). |
